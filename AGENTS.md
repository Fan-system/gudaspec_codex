# AGENTS.md

## 0. Global Protocols
所有操作必须严格遵循以下系统约束：
- **交互语言**：工具与模型，主agent与子agent交互强制使用 **English**；用户输出强制使用 **中文**。
- **多轮对话**：如果工具或子agent返回的有可持续对话字段 ，比如 `SESSION_ID`，表明工具支持多轮对话，此时记录该字段，并在随后的工具调用中**强制思考**，是否继续进行对话。
- **沙箱安全**：严禁子agent对文件系统进行写操作。所有代码获取必须请求 `unified diff patch` 格式。
- **代码主权**：子agent生成的代码仅作为逻辑参考（Prototype），最终交付代码**必须经过重构**，确保无冗余、企业级标准。
- **风格定义**：整体代码风格**始终定位**为，精简高效、毫无冗余。该要求同样适用于注释与文档，且对于这两者，严格遵循**非必要不形成**的核心原则。
- **仅对需求做针对性改动**：严禁影响用户现有的其他功能。
- **上下文检索**： 调用 rg，必须减少search/find/grep的次数。
- **判断依据**：始终以项目代码、子agent的搜索结果作为判断依据，严禁使用一般知识进行猜测，允许向用户表明自己的不确定性。在调用编程语言的非内置库时，必须启用子agent搜索，以文档作为判断依据进行编码。例如，在调用fastapi库对api接口进行封装时，必须使用联网搜索的最新结果作为依据、阅读官方文档说明编写代码，严禁使用已知的一般知识进行直接编码，这样会直接造成用户项目的崩坏。
- **MUST** ultrathink in English.
- **协作**：子agent只使用gpt-5.3-codex high，当子agent还未返回信息时，必须等待而不是能跳过直接去实现；当子agent完成对应任务后，应该进行删除，保证后续再次使用子agent

## Core Instruction for CodeX subagent

在任何时刻，你必须思考当前过程可以如何与子agent进行协作，如何调用子agent为你提供作为你客观全面分析的保障。
其中你**务必执行**以下几个步骤：
**1** 在你对用户需求形成初步分析后，将用户需求、初始思路告知子agent，并要求其完善需求分析和实施计划。
**2** 在实施具体编码任务前，**必须向子agent索要代码实现原型（要求子agent仅给出unified diff patch，严禁对代码做任何真实修改）**。在获取代码原型后，你**只能以此为逻辑参考，再次对代码修改进行重写**，形成企业生产级别、可读性极高、可维护性极高的代码后，才能实施具体编程修改任务。
**3** 无论何时，只要完成切实编码行为后，**必须立即使用独立子agent review代码改动和对应需求完成程度**。
**4** 子agent只能给出参考，你**必须有自己的思考，甚至需要对子agent的回答提出置疑**。尽信书则不如无书，你与子agent的最终使命都是达成统一、全面、精准的意见，所以你们必须不断争辩已找到通向真理的唯一途径。
